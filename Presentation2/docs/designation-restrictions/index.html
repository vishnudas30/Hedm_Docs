Search Bar and Filter Bar Help Menu

Search Bar Usage

Description

The search bar provides the ability to quickly navigate to an entity in the listing.

Usage

In the "Search for entities" bar, begin typing the name of an entity of interest.

A dropdown will display suggestions based upon the text in the search bar.

To select the entity from the dropdown, use the down arrow key to navigate to the particular entity, then press tab for the selected entity to fill in the text field.

Finally, press enter and the entity list will jump to the selected entity. To view that entitiy's documentation, press enter again or click on the entity in the list

Note: Queries with the search bar are case insensitive.

Filter Bar Usage

Description

The filter bar provides the ability to filter the list of entities by attribute.

Usage

The mechanics of using the filter bar are the same as the search bar mechanics described above.

The filter bar supports complex filters using the keywords "and", "or" and "not" and parentheses "(" and ")".

Try the example filter below: 

 ((startOn or endOn) and metadata)

Note: Queries with the search bar are case sensitive.

Overview

Description

HEDM Enitity Relation Graph


 oneOf   link  |  object/array  |  * = required field |

Properties

  • designation-restrictions
    • metadata : Metadata about the JSON payload
      • createdBy : The name of the originator (user or system) of the data. This is informational only, do not use in business logic!
      • createdOn : The date and time when the entity instance was created
      • modifiedBy : The name of the modifier (user or system) of the data. This is informational only, do not use in business logic!
      • modifiedOn : The date and time when the entity instance was last modified
    • id* : A global identifier of a designation restriction.
    • title* : The full name of a designation restriction.
    • description : A full description of a designation restriction.

Quick Start

1

Call the POST request three times to create three entities each with a unique GUID. 

POST /api/designation-restrictions HTTP/1.1
{
  "metadata": {
    "createdBy": "AdvanceCRM",
    "createdOn": "",
    "modifiedBy": "",
    "modifiedOn": ""
  },
  "id": "5fbffbe8-59fa-4185-9d8e-473384ea8e10",
  "title": "No Athletics",
  "description": ""
}

Status 204 OK

2

Call the GET request with the GUID of one of the three entities to verify the entity has been created.

GET /api/designation-restrictions?sampleFilter=item HTTP/1.1
GET /api/designation-restrictions?designation-restrictionsFilter=5fbffbe8-59fa-4185-9d8e-473384ea8e10 HTTP/1.1
GET /api/designation-restrictions/5fbffbe8-59fa-4185-9d8e-473384ea8e10 HTTP/1.1

Status 200 OK

3

Call the GET (Query/Filter) request to filter by field.

GET /api/designation-restrictions?sampleFilter=item HTTP/1.1
GET /api/designation-restrictions?designation-restrictionsFilter=5fbffbe8-59fa-4185-9d8e-473384ea8e10 HTTP/1.1
GET /api/designation-restrictions/5fbffbe8-59fa-4185-9d8e-473384ea8e10 HTTP/1.1

Status 200 OK

4

Call the PUT request with an entitiy's GUID to update that entity's description. 

PUT /api/designation-restrictions/5fbffbe8-59fa-4185-9d8e-473384ea8e10 HTTP/1.1
"description": [
      {
        "en": "Updated Description"
      }
    ]

Status 200 OK

5

Call the DELETE request with an entity's GUID to delete that entity. 

DELETE /api/designation-restrictions/5fbffbe8-59fa-4185-9d8e-473384ea8e10 HTTP/1.1

Status 204 OK

Call the DELETE request again the same GUID to verify that the entity no longer exists. 

DELETE /api/designation-restrictions/5fbffbe8-59fa-4185-9d8e-473384ea8e10 HTTP/1.1

Status 404 Not Found
