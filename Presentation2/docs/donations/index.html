Search Bar and Filter Bar Help Menu

Search Bar Usage

Description

The search bar provides the ability to quickly navigate to an entity in the listing.

Usage

In the "Search for entities" bar, begin typing the name of an entity of interest.

A dropdown will display suggestions based upon the text in the search bar.

To select the entity from the dropdown, use the down arrow key to navigate to the particular entity, then press tab for the selected entity to fill in the text field.

Finally, press enter and the entity list will jump to the selected entity. To view that entitiy's documentation, press enter again or click on the entity in the list

Note: Queries with the search bar are case insensitive.

Filter Bar Usage

Description

The filter bar provides the ability to filter the list of entities by attribute.

Usage

The mechanics of using the filter bar are the same as the search bar mechanics described above.

The filter bar supports complex filters using the keywords "and", "or" and "not" and parentheses "(" and ")".

Try the example filter below: 

 ((startOn or endOn) and metadata)

Note: Queries with the search bar are case sensitive.

Overview

Description

The donations entity contains information associated with a charitable donation to an institution. The entity covers three primary aspects of donations - the allocation of the donation to donors, the payments associated with the donation and a reference to any existing pledges that the donation fulfills.

The institutional credit for each donation can be allocated across multiple donors. For each allocation, the donor is identified (either a person or an organization) along with the amount and type of credit awarded. The type of credit is defined at a high level as either 'hard' or 'soft' credit plus a type of herd/soft credit. An optional detail credit type can also be specified. The designated use of the donation is included as part of the allocation and if a campaign or appeal is associated with the allocation, it is identified. The type of donor is specified (e.g. 'alumni', 'corporation') along with the relationship between the donor and the donation (e.g. 'Primary', 'In memory of').

There can be any number of donor allocations for a donation, but there must be at least one allocation and one of the donor allocations must identify the primary donor.

Multiple payments can be specified for each donation. For each payment, the payor (either a person or an organization) is specified with the amount and date of the payment. The payment method (e.g. 'check') is identified along with reference information (e.g. check number) as appropriate. At least one payment must be specified for each donation. The total donation amount will be the sum of all payment amounts.

If the donation is a fulfillment of previous pledges (or other promised payments), each promised donation and the amount allocated to it is identified.
