Search Bar Usage

Description

The search bar provides the ability to quickly navigate to an entity in the listing.

Usage

In the "Search for entities" bar, begin typing the name of an entity of interest.

A dropdown will display suggestions based upon the text in the search bar.

To select the entity from the dropdown, use the down arrow key to navigate to the particular entity, then press tab for the selected entity to fill in the text field.

Finally, press enter and the entity list will jump to the selected entity. To view that entitiy's documentation, press enter again or click on the entity in the list

Note: Queries with the search bar are case insensitive.

Usage

The mechanics of using the filter bar are the same as the search bar mechanics described above.

Overview

Description

In HEDM, the persons entity is used to structure the basic demographic information of the people associated with higher education. People fulfill many different (and often, multiple) roles within higher education. This entity only contains properties that are common to all roles. Properties associated with the roles people fill will be contained in role specific entities (e.g. students, constituents, ...) which will be developed and added to HEDM as required.

The persons entity includes the following properties associated with the person:

  • a list of names, by type ('legal', 'birth'),
  • the date of birth (and death, if applicable),
  • gender,
  • ethnicity,
  • race,
  • marital status,
  • a list of roles with associated dates ( e.g. 'student', 'faculty', ...)),
  • a list of credentials (e.g. SSN, system IDs) and the dates when valid,
  • a list of locations (i.e. addresses),
  • a list of phone numbers,
  • a list of email addresses,
  • a list of social media addresses.

The locations specified for a person include the address lines, address type ('home', 'school', ...), the status of the address ('active', 'inactive') and if the address should be considered the 'Primary' address of the specified type. In addition, the location can include latitude and longitude and a definition of seasonal occupancy. Each location can be assigned to multiple geographic regions/sub-regions for reporting purposes. The mailing address for each location can also be specified in a country specific format.
